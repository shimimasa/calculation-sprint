# ADR-010: 品質ゲートの役割分離とCIの無依存化

Status: Accepted
Date: 2026-02-10

## Context

- 品質ゲートが追加ツール依存を持つと、CI環境での安定運用が難しくなる。
- ローカル開発では補助ツールを使った追加チェックが有効だが、CIでは標準環境のみで完結させたい。

## Decision

- gate は開発者向けの補助チェックを許可する。
- gate-ci はCI専用の実行経路とし、Node.js標準のみで完結させる。
- CIゲートは追加ツール依存を持たない。

## Consequences

- CIではツール追加なしで確実に品質ゲートが通る。
- ローカルではより手厚いチェックを任意で実行できる。
